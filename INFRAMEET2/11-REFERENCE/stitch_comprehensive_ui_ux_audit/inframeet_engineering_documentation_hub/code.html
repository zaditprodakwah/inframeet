Core API Reference

Every request to the INFRAMEET backbone must include a valid cryptographic token in the header.

PARAMETER TYPE DESCRIPTION
cluster_id UUID_V4 The unique identifier for the target compute cluster.
entropy_seed STRING_64 Seed for localized encryption layer generation.
sync_mode ENUM Options: [STRICT, PARTIAL, ASYNC]. Default is STRICT.
INITIALIZATION_SCRIPT.TS Verified by SHA-256 
import { InfraMeet } from '@inframeet/core';

// Initialize the secure gateway with cryptographic identity
const hub = new InfraMeet({
  apiKey: process.env.INFRA_KEY,
  region: 'eu-west-secure-1',
  securityLevel: 'fortress'
});

async function deployCluster() {
  const node = await hub.provision({
    capacity: 'high_density',
    encryption: 'AES_256_GCM'
  });
  console.log(`Node deployed: ${node.id}`);
}
